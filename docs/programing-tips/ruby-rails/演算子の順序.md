# 演算子の順序

以下の優先順位によって処理される。

```txt
高い   ::
       []
       +(単項)  !  ~
       **
       -(単項)
       *  /  %
       +  -
       << >>
       &
       |  ^
       > >=  < <=
       <=> ==  === !=  =~  !~
       &&
       ||
       ..  ...
       ?:(条件演算子)
       =(+=, -= ... )
       not
低い   and or
```

```ruby
# 例
a && b || c   #=> (a && b) || c
a || b && c   #=>  a || (b && c)
```

## 短絡評価

- `||`, `&&`, `or`, `and` は短絡評価され、必要な場合だけ、右辺が評価される。

```ruby
# 例
m = true
m or n = true
p n  #=> nil (or の右辺が実行されないため)
```
